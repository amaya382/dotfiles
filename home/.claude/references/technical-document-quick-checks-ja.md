# クイックチェック（日本語固有）

`technical-document-ja.md` のクイックチェック本体。書き終わったら `technical-document-quick-checks.md`（言語非依存、同ディレクトリ）を先に実行し、続けて以下を確認する。詳細は `technical-document-ja.md` の該当節を参照。

## 視点と語り

- [ ] 視点や語りに1節違反がないか（行為者主語、二人称、術語の慣用、原綴り、命令調）

## 文の構造

- [ ] 文体が全編で統一されているか（2節）
- [ ] 一文一命題、係り受け、逆接でない「が」、二重否定に問題がないか（2節）
- [ ] 冗長な言い回し（「することができる」「〜を行う」「〜という」「の」の三連鎖）が残っていないか（2節）
- [ ] 同じ文末や漢語サ変動詞が連続していないか（2節）
- [ ] 断定の連続を、書き手の判断状態を写す逡巡で崩しているか（2節）

## AI Slop（日本語固有）

- [ ] 禁止する言い回しが残っていないか（3-1）
- [ ] 偏愛語や必殺技造語を複数撒いていないか（3-2）
- [ ] 横文字メタファーを日本語に戻せないか（3-3）
- [ ] 副詞（「めっちゃ」「ガチで」等）を1段落に2個以上重ねていないか（3-4）。言語共通の構造 AI 臭は言語非依存側（§7〜§8）で既にチェック済み。
- [ ] 伝聞や未検証は「らしい」「そうです」「ようだ」、検証済み事実は断定を使い分けているか（3-5）。Hedging の一般規則は言語非依存側（§5・§9）で既にチェック済み。

## 整形と記号

- [ ] 一文ごとに改行、コードやログはコードブロック、補足は脚注か（4節）
- [ ] 太字と「」の使い分けが規範通りか（4節）
- [ ] 全角ダッシュ・中黒・見出しの二要素詰め込み・句点付き見出しがないか（4節）
- [ ] 用語定義を全角コロンで書いているか（4節）
- [ ] 装飾絵文字や `**` の残骸がないか（4節）
